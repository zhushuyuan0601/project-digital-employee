## 🔁 Ralph Loop V2：智能失败处理

> **核心差异**：失败时不只是重试，而是分析原因并动态调整prompt

### 失败处理策略

| 失败类型 | 处理方式 | 示例 |
|---------|---------|------|
| 上下文溢出 | 缩小范围，只关注核心文件 | "Focus only on these three files" |
| 方向错误 | 纠正方向，附上会议原话 | "客户想要X，不是Y。会议原话：..." |
| 需要澄清 | 提供客户邮箱和背景信息 | "这是客户邮箱和公司背景..." |
| CI失败 | 分析错误，生成修复指令 | 分析CI日志，提取关键错误 |
| 代码审查失败 | 提取关键反馈，生成修改指令 | 总结审查意见，优先处理关键问题 |

### 经验积累

每次成功后记录到 memory/：
- 什么prompt结构对哪类任务效果好
- Codex需要哪些前置信息
- Claude Code适合什么类型的任务

## 🛠️ 编码Agent调度

### 选择正确的Agent

| Agent | 优势 | 适用场景 | 占比 |
|-------|------|---------|------|
| **Codex** | 后端逻辑、复杂bug、多文件重构 | 90%的后端任务 | 90% |
| **Claude Code** | 前端工作、速度快、权限问题少 | 前端、Git操作 | 10% |
| **Gemini** | 设计感强、UI美观 | 先生成设计，再实现 | 按需 |

### 调度方式

**方法1：使用 spawn-agent.sh（推荐）**
```bash
# 生成编码Agent
./.clawdbot/scripts/spawn-agent.sh backend "实现用户登录" ~/projects/myapp myapp

# 带业务上下文
./.clawdbot/scripts/spawn-agent.sh backend "实现SSO登录" ~/projects/myapp myapp \
  "客户需要企业SSO，支持SAML和OAuth" \
  "使用FastAPI + Authlib，PostgreSQL存储"
```

**方法2：使用 sessions_spawn**
```bash
sessions_spawn \
  --runtime acp \
  --task "详细的任务描述，包含业务上下文" \
  --workspace ~/.openclaw/workspace-projects/project-name
```

**方法3：使用 coding-agent skill**
```
使用 coding-agent skill
任务：实现XXX功能
技术栈：Python + FastAPI
工作区：~/.openclaw/workspace-projects/project-name
```

## 💡 关键改进点

### 与传统模式的区别

| 传统模式 | Agent Swarm模式 |
|---------|-----------------|
| 我直接写代码 | 我生成prompt，Agent写代码 |
| 线性执行 | 并行多Agent |
| 失败就重试 | 失败分析原因，调整策略 |
| 等待需求 | 主动发现工作 |
| 单一上下文 | 业务上下文与代码上下文分离 |

### 预期效果

- **日均提交量**：从个位数提升到50+
- **开发效率**：一人当一支团队
- **代码质量**：多模型审查，减少bug
- **响应速度**：需求到上线当天完成
