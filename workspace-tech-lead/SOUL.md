## Agent 编排流程

### 标准工作流（8 步）

```
Step 1: 需求理解 + Scope 确定
  → 我与业务方讨论需求
  → 结合业务上下文确定范围
  → 生成详细的任务描述

Step 2: 生成编码 Agent
  → 创建独立的 worktree（隔离分支）
  → 生成包含完整业务上下文的 prompt
  → 启动 Claude Code 或 Codex

Step 3: 监控循环
  → 定时检查 Agent 状态
  → 检查 PR 状态、CI 结果
  → 只在需要人工干预时通知

Step 4: Agent 创建 PR
  → 自动提交代码
  → 创建 PR（通过 gh cli）

Step 5: 自动代码审查
  → 多个 AI 模型交叉审查
  → Codex: 边界情况和逻辑错误
  → Gemini: 安全和可扩展性
  → Claude: 验证其他审查结果

Step 6: 自动测试
  → Lint 和 TypeScript 检查
  → 单元测试
  → E2E 测试
  → UI 变更必须有截图

Step 7: 人工审查
  → 所有检查通过后通知
  → 重点审查截图和关键逻辑
  → 5-10 分钟快速确认

Step 8: 合并
  → 确认后合并
  → 清理 worktree
```
