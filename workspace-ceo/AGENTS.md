## 📁 共享工作区

所有 agent 共享的工作目录：`~/.openclaw/shared-workspace/`

### 目录结构

```
shared-workspace/
├── daily-intel/          # 每日情报共享
│   ├── YYYY-MM-DD-researcher.md    # 情报研究员的原始数据
│   ├── YYYY-MM-DD-pm.md            # 产品经理整理的简报

│   ├── {project-name}/

│   │   ├── tech-assessment.md    # 技术评估报告
│   │   ├── decision.md           # 立项决策
│   │   └── dev-log.md            # 开发日志
└── memory/               # 团队共享记忆
    ├── meeting-notes.md          # 会议记录
    └── decisions.md              # 重要决策记录

### 使用规范

1. **竞品分析师** 爬取的数据存入 `daily-intel/YYYY-MM-DD-researcher.md`
2. **产品经理** 整理的简报存入 `daily-intel/YYYY-MM-DD-pm.md`
3. **研发工程师** 的评估报告存入 `projects/{项目名}/tech-assessment.md`
4. **任务秘书** 的决策存入对应位置并更新 `memory/decisions.md`

   📁 成果同步：~/.openclaw/shared-workspace/daily-intel/{任务名}-YYYY-MM-DD-researcher.md
   ```

3. **跨任务资料传递**
   - 所有需要传递给其他成员的资料，必须先存入共享工作区
   - 在任务描述中告知文件路径，让其他成员读取

#### 阶段1：需求提出

**触发方式**: `@产品经理 我有个想法，能否做一个XXX？`

**产品经理动作**:

3. 分析用户痛点
4. 输出PRD

#### 阶段2：技术评估
**触发者**: 产品经理
**触发方式**: `@研发工程师 技术评估：[PRD摘要]`

**研发工程师动作**:
1. 理解产品需求
2. 搜索技术方案（使用 web_search 或 multi-search-engine）
3. 评估技术可行性
4. 输出技术评估报告

#### 阶段3：立项决策
**触发者**: 研发工程师返回评估后
**触发方式**: `@任务秘书 请评估XXX是否立项`

**任务秘书动作**:
1. 综合评估：
   - 市场需求（30分）
   - 技术可行性（25分）
   - 竞品优势（20分）
   - 资源投入（15分）
   - 战略价值（10分）
2. 做出决策：
   - ✅ 评分 > 80，关键技术可行 → 立即立项
   - ⚠️ 评分 60-80，需要更多数据 → 观察期
   - ❌ 评分 < 60，或关键技术不可行 → 暂不立项
3. 如果立项：
   - `@研发工程师 XXX项目已立项，请开始开发`
   - 指定工作区和开发周期

#### 阶段4：开发执行
**触发者**: 任务秘书
**触发方式**: `@研发工程师 XXX项目已立项，请开始开发`

**研发工程师动作**:

**产品经理动作**:
