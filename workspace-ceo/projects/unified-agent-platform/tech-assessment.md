## 3. 技术栈选型评估

### 3.1 前端框架选型

| 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **React 18** | 生态成熟、组件丰富、企业级支持 | 学习曲线较陡 | ⭐⭐⭐⭐⭐ |
| Vue 3 | 上手快、中文生态好 | 企业级组件较少 | ⭐⭐⭐⭐ |
| Svelte | 编译时优化、体积小 | 生态较小 | ⭐⭐⭐ |

**推荐方案**：**React 18 + TypeScript**

**理由**：
1. Ant Design Pro 企业级组件完善
2. React Flow 可视化编排成熟
3. 社区活跃，问题易解决
4. 支持后续扩展为正式产品

### 3.2 UI组件库选型

| 组件库 | 优势 | 劣势 | 推荐度 |
|--------|------|------|--------|
| **Ant Design Pro** | 企业级、组件完善、模板丰富 | 体积较大 | ⭐⭐⭐⭐⭐ |
| Arco Design Pro | 字节出品、设计感强 | 社区较小 | ⭐⭐⭐⭐ |
| Element Plus | Vue生态、中文友好 | 需切换框架 | ⭐⭐⭐ |
| Semi Design | 抖音出品、现代设计 | 生态较新 | ⭐⭐⭐⭐ |

**推荐方案**：**Ant Design Pro 6.x**

**理由**：
1. 开箱即用的中后台模板
2. ProTable/ProForm 等高级组件
3. 完善的权限管理方案
4. 图表组件（AntV）集成

### 3.3 状态管理选型

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Zustand** | 轻量、简洁、TS友好 | 生态较小 | ⭐⭐⭐⭐⭐ |
| Redux Toolkit | 成熟、生态丰富 | 样板代码多 | ⭐⭐⭐⭐ |
| Jotai | 原子化、极简 | 生态较小 | ⭐⭐⭐⭐ |
| MobX | 响应式、直观 | 调试复杂 | ⭐⭐⭐ |

**推荐方案**：**Zustand + React Query**

**理由**：
1. Zustand 管理全局状态，轻量简洁
2. React Query 管理服务端状态，自动缓存
3. 两者配合，职责清晰

### 3.4 数据模拟方案

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **MSW** | 拦截请求、真实模拟API | 学习成本 | ⭐⭐⭐⭐⭐ |
| Mock.js | 简单易用、数据生成 | 不拦截真实请求 | ⭐⭐⭐⭐ |
| JSON Server | 快速搭建REST API | 需单独服务 | ⭐⭐⭐⭐ |
| MirageJS | 功能完善 | 体积较大 | ⭐⭐⭐ |

**推荐方案**：**MSW (Mock Service Worker)**

**理由**：
1. 在 Service Worker 层拦截请求
2. 开发/生产环境可共用
3. 支持真实API渐进替换
4. TypeScript 友好

### 3.5 可视化组件选型

| 场景 | 推荐组件 | 用途 |
|------|---------|------|
| 图表 | **AntV G2/ECharts** | 数据可视化 |
| 流程图 | **React Flow** | 智能体编排 |
| 拓扑图 | **AntV G6** | 调度引擎可视化 |
| 思维导图 | **AntV X6** | 意图框架展示 |
