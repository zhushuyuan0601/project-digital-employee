## 🎯 触发方式

### 场景1：产品需求提出
**在飞书群组中@产品经理**
```
@产品经理 我有个想法，能否做一个AI代码助手？
```

**响应流程：**
1. 产品经理（PM）接收需求
2. PM分析需求，输出PRD
3. PM@技术总监，请求技术评估

### 场景2：情报驱动的产品机会
**总助理主动触发**
```
@情报研究员 帮我爬取最新的AI智能体产品动态
→ 总助理分析数据
→ @产品经理 基于这些情报分析产品机会
→ @技术总监 评估技术可行性
```

### 场景3：技术驱动的创新
**技术总监主动提出**
```
@总助理 检测到XXX技术有突破，可以孵化产品
→ 总助理组织讨论
→ 产品经理提出产品方案
→ 技术总监评估实现方案
```

## 🔄 详细流程

### 阶段1：需求分析与收集

| 步骤 | 执行者 | 动作 | 输出 |
|------|--------|------|------|
| 1.1 | 青峰/外部 | 提出产品想法 | 需求描述 |
| 1.2 | **产品经理** | 需求分析，输出PRD | 产品需求文档 |
| 1.3 | **情报研究员** | 市场数据收集 | 竞品分析报告 |
| 1.4 | **总助理（CEO）** | 需求价值评估 | 需求优先级 |

### 阶段2：技术可行性评估

| 步骤 | 执行者 | 动作 | 输出 |
|------|--------|------|------|
| 2.1 | **产品经理** | @技术总监 "技术话术：[PRD摘要]" | 评估请求 |
| 2.2 | **技术总监** | 技术可行性分析 | 技术方案 |
| 2.3 | **技术总监** | 架构设计、技术选型 | 架构方案 |
| 2.4 | **技术总监** | 工作量预估 | 开发计划 |

### 阶段3：立项决策

| 步骤 | 执行者 | 动作 | 输出 |
|------|--------|------|------|
| 3.1 | **总助理（CEO）** | 综合评估（市场+技术+资源） | 立项决策 |
| 3.2 | **总助理（CEO）** | 优先级排序 | 开发队列 |
| 3.3 | **总助理（CEO）** | 资源分配 | 项目分配 |

### 阶段4：开发执行

| 步骤 | 执行者 | 动作 | 输出 |
|------|--------|------|------|
| 4.1 | **技术总监** | 使用 Claude Code 开发 | 代码实现 |
| 4.2 | **技术总监** | 协调开发 Agent | 功能实现 |
| 4.3 | **产品经理** | 产品验收 | 验收报告 |
| 4.4 | **总助理（CEO）** | 上线决策 | 上线计划 |

## 💬 飞书群组协作规范

| 产品经理 | @技术总监 | 请求技术评估 |

| 总助理 | @情报研究员 | 市场数据收集 |

### 群组角色

- **总助理（Zoe）🐾**: 群主，协调所有流程，最终决策者
- **产品经理（Sarah）🏆**: 需求收集、PRD、产品验收
- **技术总监（Alex）💻**: 技术评估、架构设计、开发执行
- **情报研究员🔍**: 市场数据、竞品分析
- **青峰**: CEO，提出需求、审批决策

## 📊 立项评估标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 市场需求 | 30% | 是否有真实用户需求 |
| 技术可行性 | 25% | 是否能实现、成本如何 |
| 竞品优势 | 20% | 与竞品相比有何优势 |
| 资源投入 | 15% | 开发时间、人力成本 |
| 战略价值 | 10% | 是否符合公司方向 |

## 🎯 立项决策矩阵

| 决策 | 标准 |
|------|------|
| ✅ 立即立项 | 评分 > 80，且关键技术可行 |
| ⚠️ 观察期 | 评分 60-80，需要更多数据 |
| ❌ 暂不立项 | 评分 < 60，或关键技术不可行 |

## 🔐 关键点

1. **所有产品需求必须通过群组提出**，不能私下沟通
2. **产品经理是需求入口**，所有人提需求都经由产品经理
3. **技术总监是技术评估者**，产品经理不能绕过直接分配开发任务
4. **总助理是协调者和决策者**，所有立项需要总助理同意
5. **开发必须通过 Claude Code 或 ACP**，技术总监负责协调
