### 5. 智家通通 GUI 对接方案设计与落地

**目标**: 基于智家通通调用 GUI 对接流程与接口描述，完成 SOP 方案场景接入设计

**本周进展**:
- 完成 SDK-AISF 接口文档深度分析，梳理核心接口设计：
  - **登录鉴权接口**（/acm/v1/auth）：终端身份认证，获取 AISF token，支持设备类型枚举（云手机/物理手机/算力主机等）
  - **文件上传接口**（/acm/v1/uploadfile）：支持图片、文件等内容上传，返回 fileUUID 关联后续业务请求
  - **AI 业务调用接口**（/ws/acm/v1/ai）：WebSocket 长连接，支持 token 认证、意图认知、任务分配、流式输出
- 设计智家通通与 GUI 系统的对接流程：
  - 建立 WebSocket 长连接，完成 token 鉴权
  - 用户意图识别报文构造（UserTaskManagement 类型）
  - 澄清结果报文交互（DataExchange 类型）
  - 任务终止与状态报告机制
- 明确接口参数规范：
  - Header 层：MessageType、Sender、Receiver、DeviceId、UUID、ClientSessionId 等
  - Data 层：UserInputText、UserSpecifiedAgent、TaskID 等
  - Payload 层：支持 ShowCtrl 卡片控件（Button、Text、TTS 等）
- 识别 SOP 方案接入关键点：
  - 设备类型映射（DeviceCategory：1=云手机，2=算力主机，3=物理手机）
  - 意图类型枚举（深度搜索=11，记忆问答=10）
  - 流式输出状态控制（Running/Completed）

**完成度**: 【50%】

**预计完成时间**: 2026-03-20

**下一步**: 
- 基于接口设计完成智家通通 SDK 对接开发
- 验证 WebSocket 长连接稳定性与 token 鉴权流程
- 构造典型场景用例（打车、查询等）进行端到端测试
