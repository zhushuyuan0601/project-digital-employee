# 数字员工监控中心 - 功能说明文档

> 本文档记录 2026-03-16 页面改造内容，用于工作成果展示。

---

## 一、页面概述

**访问地址**: `http://localhost:3002/digital-employee`

**功能定位**: 多Agent协作管理平台的实时数据监控中心，集中展示数字员工团队的工作状态、项目进度和工作成果。

---

## 二、核心功能模块

### 2.1 顶部导航栏

| 元素 | 说明 |
|------|------|
| Logo + 标题 | "数字员工监控中心"，副标题 "AI Digital Employee Command Center · 实时数据流" |
| 刷新按钮 | 手动刷新数据，带旋转动画效果 |
| 最后更新时间 | 显示数据最后同步时间 |
| 状态指示灯 | 绿色脉冲动画，表示系统在线 |

### 2.2 统计卡片（4个）

| 卡片 | 数据项 | 图标颜色 |
|------|--------|----------|
| 项目总数 | `totalProjects` | 蓝色 |
| 开发中 | `inProgress` | 绿色 |
| 规划中 | `notStarted` | 黄色 |
| 今日提交 | `todayCommits` | 紫色 |

### 2.3 数字员工卡片（4个Agent）

**展示内容**:
- Agent 头像、名称、角色
- 统计数据（任务数/提交数/报告数等）
- 在线状态（Active 标签）
- 当前任务
- 今日产出（文件链接列表）

**团队成员**:
| 角色 | 名称 | 职责 |
|------|------|------|
| 小 U | 任务秘书 | 统筹调度、任务分配、汇报 |
| 小开 | 研发工程师 | 技术总监&开发工程师 |
| 小产 | 产品经理 | 产品需求分析 |
| 小研 | 竞品分析师 | 市场调研 |

### 2.4 工作成果面板

**筛选功能**:
- 日期范围选择（开始/结束日期）
- 角色筛选（全部/小U/小开/小研/小产）
- 类型筛选（报告/文档/代码）
- 快捷按钮（今天/本周）

**成果卡片**:
- 文件类型图标（📊报告/📝文档/🔧代码）
- 文件名称、日期、负责人
- 预览按钮（支持 Markdown/代码等在线预览）

### 2.5 实时工作流面板

**筛选功能**:
- 日期范围选择
- 角色筛选
- 快捷按钮（今天/本周）

**工作记录**:
- 日期、时间、负责人
- 任务内容
- 状态标签（完成/进行中）

### 2.6 项目进度看板

**项目卡片信息**:
- 项目图标、名称、描述
- 阶段徽章（立项前/需求分析/技术评估/开发中/测试验收/已上线/维护期/已暂停/已取消）
- 进度条（百分比）
- 负责人、更新时间、任务数
- 已完成任务列表（绿色背景）
- 下一步计划列表（黄色背景）
- GitHub/Demo 链接

**阶段状态映射**:
| 阶段代码 | 阶段名称 | Emoji |
|---------|---------|-------|
| `idea` | 立项前 | 💡 |
| `analysis` | 需求分析 | 🔍 |
| `tech-assessment` | 技术评估 | 🔧 |
| `development` | 开发中 | 💻 |
| `testing` | 测试验收 | 🧪 |
| `deployed` | 已上线 | 🚀 |
| `maintenance` | 维护期 | 🔁 |
| `paused` | 已暂停 | ⏸️ |
| `cancelled` | 已取消 | ❌ |

## 三、本次改造内容（2026-03-16）

### 3.1 后端API改造

**文件**: `server/index.js`

**文件**: `src/views/DigitalEmployee.vue`

**优化内容**:

5. 加载状态动画优化
6. 亮色/暗色主题适配

## 四、技术架构

### 前端技术栈
- **框架**: Vue 3 (Composition API)
- **语言**: TypeScript 5.x
- **构建工具**: Vite 5.x
- **UI 组件库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4.x

### 后端服务
- **框架**: Express.js
- **端口**: 18888
- **API**: `/api/dashboard`

### 数据流
```
PROJECT_STATUS.md → 后端扫描解析 → /api/dashboard → 前端渲染
```
