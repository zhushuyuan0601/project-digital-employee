### 2.2 知识库服务 (Knowledge Base)

#### 2.2.1 数据模型
```typescript
interface KnowledgeItem {
  id: string;
  categoryId: string;
  question: string;
  answer: string;
  similarQuestions: string[];
  tags: string[];
  hitCount: number;
  createdAt: Date;
  updatedAt: Date;
}

interface Category {
  id: string;
  parentId: string | null;
  name: string;
  sortOrder: number;
}
```

#### 2.2.2 检索流程
1. 使用Elasticsearch进行全文检索
2. 语义相似度计算（向量检索）
3. 结果排序与过滤
4. 返回Top-N候选答案

## 3. API 接口设计

### 3.1 对话接口

#### 创建会话
```http
POST /api/v1/chat/sessions
Content-Type: application/json

{
  "userId": "user_12345",
  "channel": "web",
  "metadata": {
    "pageUrl": "https://example.com/product/123",
    "userAgent": "Mozilla/5.0..."
  }
}

Response:
{
  "code": 200,
  "data": {
    "sessionId": "sess_abc123",
    "createdAt": "2026-03-08T10:30:00Z"
  }
}
```

#### 发送消息
```http
POST /api/v1/chat/sessions/{sessionId}/messages
Content-Type: application/json

{
  "content": "我想查询订单状态",
  "type": "text"
}

Response:
{
  "code": 200,
  "data": {
    "messageId": "msg_xyz789",
    "reply": {
      "content": "请提供您的订单号，我可以帮您查询",
      "type": "text",
      "intent": "query_order"
    },
    "suggestions": ["查看最近订单", "联系人工客服"]
  }
}
```

## 4. 数据库设计

### 4.1 会话表 (chat_sessions)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 会话ID |
| user_id | VARCHAR(64) | 用户ID |
| status | TINYINT | 状态：0-进行中 1-已结束 2-转人工 |
| channel | VARCHAR(32) | 渠道：web/app/小程序 |
| created_at | DATETIME | 创建时间 |
| ended_at | DATETIME | 结束时间 |

### 4.2 消息表 (chat_messages)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 消息ID |
| session_id | BIGINT FK | 会话ID |
| content | TEXT | 消息内容 |
| type | TINYINT | 类型：1-文字 2-图片 3-卡片 |
| sender_type | TINYINT | 发送者：1-用户 2-机器人 3-人工 |
| intent | VARCHAR(64) | 意图类型 |
| created_at | DATETIME | 发送时间 |

## 5. 部署架构

### 5.1 容器化部署
```yaml
# docker-compose.yml
version: '3.8'
services:
  chat-service:
    image: chat-service:latest
    replicas: 3
    ports:
      - "8080:8080"
    environment:
      - DB_HOST=mysql
      - REDIS_HOST=redis
      - LLM_API_KEY=${LLM_API_KEY}
    depends_on:
      - mysql
      - redis
      - elasticsearch
```

### 5.2 监控告警
