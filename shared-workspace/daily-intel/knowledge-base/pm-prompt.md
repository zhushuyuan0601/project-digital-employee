## 双重同步规则（重要！）

**每个工作成果必须写入两个地方：**

### 0. 创建项目时（如果是新项目）

**必须创建中文名称文件：**

```
projects/{项目名}/{项目名}.txt
```

文件内容：项目中文名称（仅一行）

示例：
- 文件路径：`projects/ai-rbt-product/ai-rbt-product.txt`
- 文件内容：`AI彩铃产品`

### 1. 角色目录

```
/Users/lihzz/.openclaw/shared-workspace/daily-intel/roles/pm/YYYY-MM-DD/
```

文件命名：
```
YYYY-MM-DD_产品名称_PRD_pm.md
```

示例：
```
roles/pm/2026-03-08/2026-03-08_AI彩铃PRD_pm.md
```

### 2. 项目目录（同步）

```
/Users/lihzz/.openclaw/shared-workspace/daily-intel/projects/{项目名}/product/
```

文件命名：
```
产品名称PRD.md
```

示例：
```
projects/ai-rbt-product/product/AI彩铃PRD.md
```

---

## 为什么需要双重同步？

- **角色目录**：方便查看每个成员的工作成果
- **项目目录**：方便查看一个项目的所有资料（不会散落在多个角色目录）

---

## 示例：AI彩铃PRD

**步骤1：写入角色目录**
```
roles/pm/2026-03-08/2026-03-08_AI彩铃PRD_pm.md
```

**步骤2：同步到项目目录**
```
projects/ai-rbt-product/product/AI彩铃PRD.md
```
