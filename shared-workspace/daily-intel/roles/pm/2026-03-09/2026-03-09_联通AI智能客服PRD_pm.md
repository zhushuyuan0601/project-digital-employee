## 一、产品方案

### 1.1 产品定位

**一句话定位**: 基于元景大模型的新一代智能客服平台，融合联通助理和安全管家能力

| 维度 | 说明 |
|------|------|
| **产品形态** | H5智能客服页面 + 微信小程序 + APP |
| **目标客户** | 联通手机用户、企业客户 |
| **核心价值** | 7×24小时智能服务、统一服务入口、个性化体验 |
| **技术支撑** | 元景大模型、联通助理技术、安全管家能力 |

### 1.2 目标用户

#### 用户画像

| 用户类型 | 占比 | 特征 | 核心需求 |
|---------|------|------|---------|
| **普通用户** | 60% | 25-45岁，咨询套餐、办理业务 | 快速解决问题 |
| **老年用户** | 20% | 60+岁，防诈骗、简化操作 | 语音交互、简单易用 |
| **企业用户** | 15% | 中小企业，批量管理 | 企业服务、效率工具 |
| **家庭用户** | 5% | 有老人小孩，安全防护 | 亲情守护、上网管理 |

#### 用户旅程

```
普通用户旅程：
遇到问题 → 打开H5 → AI对话 → 问题解决

老年用户旅程：
遇到问题 → 语音唤醒 → AI语音对话 → 问题解决

企业用户旅程：
管理需求 → 登录后台 → 批量操作 → 完成管理
```

### 1.4 交互流程

#### 场景一：话费查询

```
用户进入H5
    │
    ▼
显示欢迎语 + 快捷入口
    │
    ▼
用户输入"查话费"或点击"话费查询"
    │
    ▼
AI识别意图 → 调用查询API
    │
    ▼
返回话费信息卡片
    │
    ▼
用户继续提问 或 结束对话
```

#### 场景二：故障报修

```
用户输入"网络坏了"
    │
    ▼
AI识别故障报修意图
    │
    ▼
引导用户描述问题
    │
    ├─── 简单问题 ───▶ AI提供解决方案
    │
    └─── 复杂问题 ───▶ 创建工单 → 预约上门
```

#### 场景三：防诈骗咨询

```
用户输入"这个电话是诈骗吗"
    │
    ▼
AI识别来电号码
    │
    ├─── 已知诈骗 ───▶ 警告用户，建议拉黑
    │
    └─── 正常号码 ───▶ 告知用户安全
```
