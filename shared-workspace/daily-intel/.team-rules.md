## 双重同步规则（重要！）

### 规则说明

**每个工作成果必须写入两个地方：**

1. **角色目录** - 按角色 + 日期组织
2. **项目目录** - 按项目 + 类型组织

### 为什么需要双重同步？

- **角色目录**：方便查看每个成员的工作成果
- **项目目录**：方便查看一个项目的所有资料（不会散落在多个角色目录）

---

## 文件命名规范

### 角色目录

```
roles/{角色}/YYYY-MM-DD/YYYY-MM-DD_任务名称_角色.md
```

示例：
```
roles/researcher/2026-03-08/2026-03-08_天翼智铃竞品分析_researcher.md
```

### 项目目录

```
projects/{项目名}/{类型}/任务名称.md
```

示例：
```
projects/ai-rbt-product/research/天翼智铃竞品分析.md
```

## 项目目录结构

**重要规则：禁止随意创建新项目！**

### ⚠️ 只有任务秘书（小U）可以创建新项目

**其他成员不能在 projects 根目录下随意创建新项目！**

- ❌ 错误：研发工程师在 `projects/` 下创建新项目
- ✅ 正确：研发工程师在现有项目的 `tech/` 目录下开发代码

### 项目结构示例

```
projects/
└── unicom-ai-customer-service/    # 项目目录（由小U创建）
    │
    ├── unicom-ai-customer-service.txt     # 项目中文名称文件
    │   内容：联通AI客服
    │
    ├── research/              # 竞品分析
    │   └── 天翼智铃竞品分析.md
    │
    ├── product/               # PRD
    │   └── AI客服PRD.md
    │
    ├── tech/                  # 技术方案 + 开发代码（重要！）
    │   ├── 技术架构.md
    │   ├── src/              # 源代码
    │   ├── package.json      # 依赖配置
    │   └── README.md
    │
    └── demo/                  # Demo
        └── demo设计.md
```

### 项目命名文件规则

- **文件名**：`{项目英文名}.txt`
- **文件位置**：项目目录根目录
- **文件内容**：项目中文名称（仅一行）
- **创建者**：只有任务秘书（小U）可以创建
- **示例**：
  - `ai-rbt-product.txt` → 内容：`AI彩铃产品`
  - `unicom-smart-ringtone.txt` → 内容：`联通智铃`

## 工作成果同步规范（强制）

### 规则

1. **完成任务后立即同步** - 将成果写入对应路径
2. **双重同步** - 角色目录 + 项目目录
3. **任务汇报中注明文件路径**
4. **跨任务资料传递** - 先存入共享工作区
5. **禁止随意创建新项目** - 只有任务秘书（小U）可以创建项目
6. **开发代码放在项目的 tech/ 目录** - 不要在 projects 根目录下创建新项目

### 同步路径

| 角色 | 角色目录 | 项目目录 |
|------|---------|---------|
| 小研 🔍 | `roles/researcher/YYYY-MM-DD/` | `projects/{项目名}/research/` |
| 小产 🏆 | `roles/pm/YYYY-MM-DD/` | `projects/{项目名}/product/` |
| 小开 💻 | `roles/tech-lead/YYYY-MM-DD/` | `projects/{项目名}/tech/` |
| 小U 🐾 | `roles/ceo/YYYY-MM-DD/` | `projects/{项目名}/` |
