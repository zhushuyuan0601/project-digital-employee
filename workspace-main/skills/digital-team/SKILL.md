# Digital Team - 数字员工团队协作系统

## 概述

这是一个模拟"数字员工团队办公"的演示系统，展示OpenClaw多Agent协作能力。

## 工作流程

```
领导
 ↓
任务秘书（小U/Zoe）
 ↓
竞品分析师（小研/researcher）
 ↓
产品经理（小产/pm）
 ↓
研发工程师（小开/tech-lead）
 ↓
ClaudeCode工程团队
 ↓
Demo生成
```

## 数字员工角色

### 1. 小U（任务秘书）- 对应现有Zoe智能体
**职责：**
- 接收领导的自然语言任务
- 与领导确认需求
- 将任务拆解为子任务
- 分配给不同数字员工
- 跟踪任务执行状态
- 最终汇总结果并向领导汇报

**成员协调：**
- 小研 - 竞品分析师
- 小产 - 产品经理
- 小开 - 研发工程师

### 2. 小研（竞品分析师）- 对应现有researcher智能体
**职责：**
- 调研行业产品
- 分析竞品能力
- 总结技术路线
- 输出结构化竞品报告

**分析维度：**
- 产品定位
- 核心功能
- 技术能力
- 用户规模
- 运营数据
- 优势与不足

### 3. 小产（产品经理）- 对应现有pm智能体
**职责：**
- 根据竞品分析设计产品方案
- 明确产品定位
- 设计核心功能
- 设计用户流程
- 输出产品需求

### 4. 小开（研发工程师）- 对应现有tech-lead智能体
**职责：**
- 根据产品方案设计技术架构
- 制定技术实现路线
- **调度ClaudeCode工程团队**
- 生成Demo方案

**关键规则：**
- 一个ClaudeCode实例负责一个开发项目
- 可以调度多个ClaudeCode并行完成不同开发任务
- 最终生成可演示的Demo

## 演示脚本

### 场景：AI智能彩铃产品

**领导指令：**
> 最近我看到中国电信上线了"天翼智铃"。用户可以用AI自动生成视频彩铃。我们联通能不能也做一个类似的能力？

**执行流程：**

1. **Zoe接收任务** → 拆解为4个子任务
2. **小研调研** → 天翼智铃竞品分析
3. **小产设计** → AI彩铃工坊产品方案
4. **小开设计** → 技术架构 + 分配ClaudeCode开发任务
5. **ClaudeCode开发** → 生成Demo
6. **Zoe汇总** → 向领导汇报

## 演示时长

- 完整演示：10分钟
- 核心流程：3-5分钟

## 真实协作特点

1. **并行工作** - 多个数字员工同时执行任务
2. **信息流转** - 小研→小产→小开，完整信息链
3. **工程落地** - 最终调度ClaudeCode实际开发
4. **进度监控** - 实时跟踪每个员工的工作状态

## 技术实现

- 使用 `sessions_spawn` 创建数字员工会话
- 使用 `sessions_send` 跨会话通信
- 使用 `coding-agent` skill 调度ClaudeCode
- 使用WebSocket实现演示页面的实时更新
