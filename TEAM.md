## 快速开始

### 与墨言对话（材料撰写）
```
"墨言，这是本季度三个核心数据：A 增长 20%，B 完成 15 万，C 用户留存达 85%。
请根据我之前的年报风格，起草一份部门季度总结简报，重点突出增长韧性。"
```

### 与纬策对话（解决方案）
```
"纬策，这是我们的新产品 API 文档和当前架构图。
现有客户需求是希望实现实时数据可视化。
请结合我们的技术优势，设计一套低延迟、可扩展的实现方案。"
```

### 与元栈对话（技术实现）
```
"元栈，我们需要一个自动收集服务器日志并发送告警的机器人。
请设计技术方案，用 Python 实现核心代码，并列出部署步骤。"
```

### 与览迹对话（知识管理）
```
"览迹，我今天部署 OpenClaw 时，在权限配置步骤卡了 2 小时，
最后发现是 SELinux 没关。这是详细过程和命令。
请记录，并生成一份可供团队参考的部署检查清单。"
```

### 典型协作场景

**场景 1：新项目开发**
1. `青峰` → `纬策`：设计解决方案
2. `纬策` → `元栈`：技术实现评估
3. `元栈` → 执行开发（可调度 Claude Code）
4. `览迹`：记录技术决策和架构文档

**场景 2：技术汇报材料**
1. `青峰` → `墨言`：撰写汇报材料
2. `墨言` → `览迹`：查询历史技术文档
3. `墨言` → `元栈`：确认技术细节
4. `墨言`：输出完整汇报 PPT

**场景 3：问题解决**
1. `青峰` → `览迹`：记录问题和解决方案
2. `览迹`：检索是否有类似历史问题
3. `览迹`：生成教程文档

### 代理间通信

已启用 `sessions_send`、`sessions_spawn` 等工具，支持：
- 代理间消息传递
- 子任务委派
- 历史会话查询

### 知识库共享

所有代理都可以访问 `览迹` 的知识库：
- 通过 `memory_search` / `memory_get` 检索
- 通过 `feishu_wiki` 访问飞书知识库
- 通过 `feishu_doc` 读写文档

## 培养建议

### 墨言的培养
1. 提供你过往满意的报告、PPT 作为范文
2. 每次给出具体反馈（如"结尾要更有号召力"）
3. 逐步建立你的写作风格模型

### 纬策的培养
1. 逐步"投喂"产品文档、技术栈介绍
2. 提问式训练："基于我们的技术栈，如何实现 X 功能？"
3. 记录每次方案的实施效果

### 元栈的培养
1. 从简单脚本开始，逐步到系统设计
2. 让它提供技术选型建议并解释理由
3. 建立代码风格和技术规范

### 览迹的培养
1. 首个建议建立的代理
2. 随时记录遇到的问题和解决过程
3. 定期整理成教程和检查清单

## 使用技巧

1. **明确称呼**: 直接叫名字（墨言、纬策等）开始对话
2. **提供上下文**: 越详细的信息，输出越精准
3. **及时反馈**: 帮助代理学习和改进
4. **善用协作**: 复杂任务让代理之间互相配合
